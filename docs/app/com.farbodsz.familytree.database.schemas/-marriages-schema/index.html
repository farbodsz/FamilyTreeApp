MarriagesSchema

object MarriagesSchema

Schema for the 'marriages' table. It contains constants for the column names of the table, and SQLite statement for creation.

This table does not contain a primary key: a composite primary key from the two ID columns would not work if the same two people re-married; and a separate ID referring to the marriage would have no purpose. Hence, there is no primary key and the correct row can be identified by using the two ID columns present.
